# タスク完了時のチェックリスト

## 必須実行項目

### 1. コード品質チェック
```bash
cd frontend

# フォーマット実行
pnpm format

# リント実行
pnpm lint
```

### 2. テスト実行
```bash
cd frontend

# 全テスト実行
pnpm test

# カバレッジ確認（推奨）
pnpm test:coverage
```

### 3. 型チェック
```bash
cd frontend

# TypeScript型チェック
npx tsc --noEmit
```

### 4. ビルド確認
```bash
cd frontend

# プロダクションビルド
pnpm build
```

## TDD サイクル確認

### RED フェーズ
- [ ] テストを書いた
- [ ] テストが失敗することを確認した
- [ ] 失敗理由が明確である

### GREEN フェーズ
- [ ] 最小限の実装でテストを通した
- [ ] 全テストがパスすることを確認した
- [ ] 過剰な実装をしていない

### REFACTOR フェーズ
- [ ] コードを改善した
- [ ] テストが引き続きパスすることを確認した
- [ ] 重複を削除した
- [ ] 可読性を向上させた

## コミット前チェック

### コード品質
- [ ] 不要なコメントアウトを削除
- [ ] console.logを削除（デバッグ用）
- [ ] 未使用のインポートを削除
- [ ] 型エラーがない
- [ ] リントエラーがない

### テスト
- [ ] 新機能にテストを追加
- [ ] 既存テストが全てパス
- [ ] プロパティベーステストが100回以上実行
- [ ] エッジケースをカバー

### ドキュメント
- [ ] 必要に応じてREADMEを更新
- [ ] 複雑なロジックにコメント追加
- [ ] 型定義にJSDocを追加（必要に応じて）

### Git
- [ ] コンベンショナルコミット形式を使用
  - `feat:` 新機能
  - `fix:` バグ修正
  - `test:` テスト追加・修正
  - `refactor:` リファクタリング
  - `docs:` ドキュメント更新
  - `chore:` その他の変更
- [ ] コミットメッセージが明確
- [ ] 原子的なコミット（1つの変更に焦点）

## 品質基準

### 最低限の基準
- テストカバレッジ: 80%以上
- 型安全性: strictモードでエラーなし
- リント: エラーなし
- ビルド: 成功

### 推奨基準
- パフォーマンス: Lighthouse スコア 90以上
- アクセシビリティ: WCAG AA準拠
- セキュリティ: 脆弱性なし（pnpm audit）
- バンドルサイズ: 初期バンドル 200KB以下（gzip）

## トラブルシューティング

### テスト失敗時
1. エラーメッセージを確認
2. 問題を再現できる最小ケースを作成
3. デバッガーで調査
4. 修正後、全テストを再実行

### ビルド失敗時
1. 型エラーを確認（`npx tsc --noEmit`）
2. 依存関係を確認（`pnpm install`）
3. キャッシュをクリア（`.next` ディレクトリ削除）
4. 再ビルド

### リント/フォーマットエラー時
1. 自動修正を試す（`pnpm format`）
2. 手動で修正が必要な箇所を確認
3. 修正後、再度チェック
